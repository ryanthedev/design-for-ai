description: Establish design foundations — purpose, audience, aesthetic direction
---

Run the design-for-ai skill in setup mode. Gather design context through conversation:

1. Ask: "What are you building? What problem does it solve?"
2. Ask: "Who is the audience? Describe a typical user."
3. Ask: "Pick 3 words for the personality/feeling of this design."
4. Ask: "Any brand constraints? Framework? Accessibility requirements?"
5. Ask: "Show me a reference site (what should it feel like?) and an anti-reference (what should it NOT feel like?)."
6. Read references/chapter-01-why-design-matters.md and references/chapter-02-purpose-of-design.md for grounding.
7. Synthesize into a design context block and offer to save it to the project's CLAUDE.md.

Output a summary: purpose statement, audience, aesthetic direction, constraints, and suggested next steps ("/fonts to choose typography, /color to build a palette, or /design-for-ai to run the full APPLIER workflow").
